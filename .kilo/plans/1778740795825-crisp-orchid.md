## 执行步骤

### 1. 暂存所有文件
```bash
git add -A
```
将所有修改和新文件加入暂存区。注意 `src/Mamba/model/mamba` 是子模块，`git add` 会更新子模块指针。

### 2. 提交更改
```bash
git commit -m "chore: add TCN model results and update project files"
```

### 3. 推送到 ModernTCN_v1 远程仓库
```bash
git push -u ModernTCN_v1 main
```
使用 `-u` 参数设置上游跟踪为 `ModernTCN_v1/main`，后续可直接 `git push`。

## 注意事项

- `.kilo/kilo.jsonc` 是 Kilo 的本地配置文件，会一并提交。如果不想提交可后续加入 `.gitignore`。
- `src/Mamba/model/mamba` 是 git 子模块，当前显示 "modified content"（子模块内部有本地修改）。`git add` 会更新子模块 commit 指针，但子模块内部的修改不会被包含。
- `.mat` 模型文件较大，确认 ModernTCN_v1 仓库没有文件大小限制（GitHub 单文件限制 100MB）。当前文件大小需确认。
