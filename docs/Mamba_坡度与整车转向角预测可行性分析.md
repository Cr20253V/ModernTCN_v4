# Mamba 预测坡度角与整车转向角方案可行性分析

日期：2026-03-25  
适用范围：`src/Mamba/Mamba_gen_train_data.m` 及路径生成链路

## 1. 结论

该方案可行，且与当前工程方向一致：

1. Mamba 预测坡度角（theta_ground）已经具备数据基础。
2. 将“两个驱动轮转向信息”统一为“整车等效转向角”作为目标是合理的，更符合控制与部署语义。
3. 在数据集生成阶段动态修改目标路径中的坡度与转弯区转向强度，可显著提升模型对工况变化的泛化能力。
4. 没有发现明显结构性冲突；主要风险集中在标签定义一致性与低速数值稳定性，均可通过规则约束解决。

## 2. 目标定义建议

### 2.1 坡度角目标

建议目标：整车地面坡度角 `theta_ground`（连续回归）。

说明：该目标与已有训练链路一致，且有明确物理意义，适合用于状态估计与补偿。

### 2.2 整车转向角目标（重点）

建议将目标定义为“整车等效转向角” `delta_vehicle`，而不是左右驱动轮分别角度。

推荐定义（运动学等效）：

- 曲率：`kappa = omega_z / max(v, v_eps)`
- 等效转向角：`delta_vehicle = atan(L * kappa)`

其中：

1. `omega_z` 为车体偏航角速度。
2. `v` 为车体纵向速度估计（可由轮速融合或已有速度通道得到）。
3. `L` 为等效轴距（采用与控制器一致的标定值）。
4. `v_eps` 建议 0.15 到 0.25 m/s，用于避免低速时数值发散。

注：训练标签可用高质量内部量构造；推理阶段仅用传感器可观测输入，不冲突。

## 3. 动态修改路径方案可行性

## 3.1 为什么需要动态修改

如果坡度与转弯强度固定，模型会学习到过窄分布，导致：

1. 对未见坡度幅值泛化较差。
2. 对转向激烈程度变化敏感，误差在高曲率段放大。
3. 坡度与转弯耦合时容易误判。

## 3.2 可行修改方向

在每次 run 中随机化以下路径参数：

1. 坡度区：坡度幅值、持续时间、进入斜率（平滑过渡时间）。
2. 转弯区：曲率或等效转向角峰值、持续时间、左右转比例、进入/退出过渡时间。
3. 坡度与转弯的相对时序：
   - 解耦样本：仅坡度或仅转弯。
   - 耦合样本：坡度与转弯重叠（建议保留约 20% 到 30%）。

这与当前注入机制并行兼容，不影响 slip/stall 注入通道。

## 4. 兼容性与风险评估

## 4.1 兼容性

1. 与当前 Mamba 造数流程兼容：可在路径生成阶段增加参数随机化，不需要改动仿真主干。
2. 与已有标签体系兼容：可新增 `delta_vehicle_target` 回归标签，同时保留分类标签（turn/slip/stall）。
3. 与“可观测输入优先”原则兼容：目标可用内部量构造，但输入仍保持传感器通道。

## 4.2 主要风险与规避

1. 低速段 `delta_vehicle` 抖动：
   - 规避：引入 `v_eps` 下限，并对标签做轻度低通或 dwell 约束。
2. 目标泄漏风险（把未来路径信息直接给输入）：
   - 规避：输入不包含未来参考轨迹，仅作为标签来源。
3. 分布失衡（极端工况过少）：
   - 规避：分层采样，单独控制坡度、转弯、耦合三类占比。

## 5. 建议的数据集标签结构（下一步实现参考）

每个样本建议包含：

1. 回归目标：
   - `y_theta_ground`
   - `y_delta_vehicle`
2. 分类目标（可选多头）：
   - `label_turn`（左/直/右）
   - `label_slip`（0/1）
   - `label_stall`（0/1）
3. 元信息：
   - run 级路径参数（坡度幅值、曲率峰值、区段起止）
   - 注入事件窗口（slip/stall）

## 6. 实施建议（小步迭代）

建议按两阶段推进：

1. 阶段 A（低风险）：
   - 在现有脚本中新增 `delta_vehicle` 目标计算与保存。
   - 保持现有场景结构，只增强参数随机化范围。
2. 阶段 B（增强泛化）：
   - 引入“坡度-转弯耦合区段”的显式采样配比。
   - 对标签分布做统计并闭环调整采样策略。

## 7. 验收标准

1. 标签稳定：低速与稳态段无异常尖峰。
2. 覆盖充分：坡度与转向目标分布无明显塌缩。
3. 可训练性：回归损失稳定下降，分类头不出现长期单类塌陷。
4. 业务一致：整车转向角定义在训练、评估、部署阶段一致。
