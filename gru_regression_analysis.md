# GRU 坡度回归结果图问题分析与改进计划（仅 slope 类评估）

日期：2026-01-07

> 目标：针对“仅在 slope 类样本上评估”的坡度回归结果图（散点图、误差直方图/CDF），总结当前结果与理想形态的差距、可能原因，以及可执行的改进路线（用于后续迭代与重新训练）。

---

## 1. 当前结果与理想结果的差距（从图形态出发）

### 1.1 理想的坡度回归结果应呈现的形态

**散点图（$\theta_{true}$ vs $\theta_{pred}$）理想特征：**
- 点云整体沿 $y=x$ 对角线分布，且不同坡度区间覆盖充分。
- 同一 $\theta_{true}$ 附近的垂直散布较小（预测方差低）。
- 上坡/下坡两侧性能相对对称，不出现明显“一侧更糟”。

**误差分布（直方图 + CDF）理想特征：**
- 误差分布以 0 为中心，近似对称；峰尖更窄、尾部更短。
- CDF 在 0 附近上升快，说明大部分样本误差较小。

### 1.2 当前图像暴露出的主要偏差类型（问题现象）

从当前生成的两张坡度回归图可见的典型问题：

1) **预测输出“带状/团簇化”**
- 散点不沿 $y=x$ 形成连续云团，而出现明显的水平带状聚集。
- 含义：回归输出动态范围被压缩，可分辨性不足，模型更像“输出到少数常见水平值”。

2) **系统性偏差（bias）**
- 误差分布不以 0 为中心且不对称，CDF 中心段偏移。
- 含义：存在整体过估或欠估趋势（或对正/负坡度不对称）。

3) **误差长尾/离群点**
- 直方图尾部较长（少量样本出现很大误差）。
- 含义：数据/标签错配、过渡段、异常噪声段、或分布外样本会产生“灾难点”。

4) **$[-2^\circ,2^\circ]$ 缺数据（在 slope-only 评估条件下）**
- 散点图中间坡度区间缺少样本点。
- 解释（更可能）：由于仅选择 slope 类样本进行评估，而 slope 类通常由标签阈值定义，近零坡度被标成 flat 并被筛掉，因此 slope-only 子集天然缺少 $[-2^\circ,2^\circ]$。
- 反例排除：`GRU_infer` 本身不包含对 $\hat\theta$ 的死区/截断后处理，因此“中间空洞”更可能来自标注/筛选而非推理死区。

## 2. 可能根因清单（按模块分类）

### A) 数据与标注（优先排查，常是最大根因）

A1. **坡度真值取值档位化（分段常值/离散坡度）**
- 现象：$\theta_{true}$ 在少数几个值附近聚集（散点出现竖条纹/集中区）。
- 影响：模型更容易学成“输出到几个档位”，导致预测带状。
- 验证信号：统计 slope 样本 $\theta_{true}$ 直方图，查看是否少峰值。

A2. **坡度覆盖不均（中等坡度/微坡度样本不足）**
- 现象：某些坡度区间误差显著更大。
- 影响：回归泛化差、误差长尾。
- 验证信号：对 $\theta_{true}$ 分箱统计样本数与 MAE（例如每 1° 一个 bin）。

A3. **窗口末端对齐导致过渡段错配（label mismatch）**
- 机制：输入窗口包含大量旧工况量测，但监督标签取窗口末端时刻，导致“输入-标签不一致”。
- 影响：产生离群点与长尾误差，尤其在坡度切入/切出阶段。
- 验证信号：离群点是否集中在工况切换附近；按“标签持续时间/驻留时间”分组统计 MAE。

A4. **噪声注入/扰动强度过大且与坡度信号混淆**
- 影响：模型倾向回归到均值，散点压缩；长尾增加。
- 验证信号：按噪声注入回合/扰动强度分组评估 MAE；查看误差分布宽度变化。

A5. **坡度符号或单位不一致（rad/deg、坐标系方向）**
- 影响：出现明显 bias，或上坡/下坡性能严重不对称。
- 验证信号：抽样核对真值/预测单位与符号约定；对正/负坡度分别统计 MAE。


### B) 模型结构与损失设计（多任务共享下常见）

B1. **回归损失权重偏小，被分类任务“压制”**
- 影响：分类很好但回归学得粗；预测被压缩到少数值。
- 验证信号：训练日志中三路损失量级差异；回归 head 梯度长期偏小。

B2. **回归只在 slope 样本计算，但 slope 样本占比低**
- 影响：有效更新稀疏，相当于“回归 batch 很小”，回归拟合不细。
- 验证信号：每个 batch 中 mask_theta=1 的比例；总体 slope 样本占比。

B3. **回归损失形式不鲁棒（对离群点敏感或尾部控制差）**
- 影响：长尾误差明显。
- 验证信号：将 MAE 改为 Huber/Charbonnier 后，尾部是否显著收回。

B4. **回归 head 过于线性，表达能力不足**
- 影响：斜率偏差、动态范围压缩。
- 验证信号：回归 head 加一层小 MLP 后，散点是否更接近 $y=x$。


### C) 训练策略与超参

C1. **早停触发时分类已收敛但回归仍未充分收敛**
- 影响：回归欠拟合，散点仍呈带状。
- 验证信号：验证集 slope-only MAE 在早停点附近是否仍在下降。

C2. **学习率/退火策略导致后期收敛不足**
- 影响：回归无法精细拟合，误差分布偏宽。
- 验证信号：延长训练/降低后期 LR 后，MAE 与散点形态是否改善。


### D) 特征与预处理

D1. **输入特征对坡度可辨识信息不足或被强噪声淹没**
- 影响：回归只能输出“近似平均值”，散点压缩。
- 验证信号：对关键特征做消融或增强（加入更多与坡度相关的派生量）后，回归是否改善。

D2. **标准化/滤波参数不一致（训练-评估不一致）**
- 影响：整体偏差或性能退化。
- 验证信号：确认推理使用的 scaler 与训练一致；在线/离线滤波常数一致。


### E) 评估口径与版本对齐

E1. **模型/数据集版本不一致导致 MAE 与论文数字不一致**
- 影响：图上 MAE 与文中报告值偏离。
- 验证信号：固定同一套 `GRU_model.mat`、`GRU_dataset_processed.mat` 与同一评估脚本口径；记录 hash/时间戳。

## 3. 可执行的改进路线（建议分阶段实施）

> 总原则：在不改变“仅 slope 类评估”的前提下，优先通过数据覆盖与训练目标对齐解决“压缩/带状 + 偏差 + 长尾”。

### Phase 0（快速诊断与版本对齐，0.5–1 天）
1. 固定并记录：模型文件版本、数据集文件版本、评估脚本版本。
2. 在 slope-only 子集上输出：
   - $\theta_{true}$ 直方图（确认是否档位化/缺区间）。
   - 分箱 MAE（每 1° 或 0.5° bin），定位哪个坡度区间最差。
   - 正/负坡度分别统计 MAE，检查对称性。

**判定标准：**
- 若 $\theta_{true}$ 档位明显 → 优先补数据覆盖；
- 若某些区间样本极少 → 优先做重采样/分层采样；
- 若正负坡度 MAE 差异大 → 排查符号/标注/特征偏置。


### Phase 1（数据侧优先，1–2 天）
1. **增加坡度幅值覆盖与连续性**
- 增加更多坡度档位或连续坡度段，尤其是模型目前表现差的区间。

2. **分层采样/重采样（只针对 slope 样本）**
- 目标：每个 batch 有足够的 slope 样本，并且不同坡度区间的样本更加均衡。

3. **降低过渡段错配样本对训练的伤害**
- 简单做法：对“标签刚切换”附近的窗口降低训练权重，或剔除一小段过渡窗口。
- 目标：收回长尾误差。


### Phase 2（损失与权重对齐，1 天内可多轮试验）
1. 提高回归损失权重（相对分类）或使用动态权重。
2. 回归损失改为鲁棒损失（Huber/Charbonnier），优先压长尾。
3. 早停指标纳入 slope-only MAE（或将其与总损失组合）。


### Phase 3（结构最小改动，2–3 天）
1. 回归 head 增强：线性 head → 小型 MLP（例如 hidden→32→1）。
2. 调整容量与正则：适当增大 hidden size、降低 dropout（若判断为欠拟合/过强正则）。


### Phase 4（闭环一致性验证，0.5–1 天）
1. 离线坡度回归改善后，验证在线延迟、抖动与闭环性能是否同步改善。
2. 重点关注：
- 是否出现“离线更准但在线抖动更大”的反效果；
- 是否需要更强的在线滤波/驻留时间配合。

---

## 4. 结论摘要（用于论文写作前的工程决策）

- 当前坡度回归图呈现“带状/团簇 + 偏差 + 长尾”，说明回归任务尚未学到充分的连续映射，可能受到数据覆盖与多任务权重的共同影响。
- $[-2^\circ,2^\circ]$ 缺样本在 slope-only 评估设定下更可能是标签阈值与筛选逻辑导致，而非推理死区。
- 推荐优先做 Phase 0/1（版本对齐 + 数据覆盖 + 采样均衡），通常收益最大；之后再做损失权重与轻量结构改动。
