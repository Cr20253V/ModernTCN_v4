# I. 对角式双舵轮 AGV 动力学与运动学模型总结

## 理想运行环境和基本假设

1. 假设 AGV 车体为绝对刚体，在重载或高动态行驶过程中不发生弹性形变；
2. 假设车辆主要在二维平面内运动，忽略垂直方向的跳动、俯仰和侧倾的动态惯性效应；
3. 假设轮胎始终垂直于地面，忽略车轮外倾角及其产生的侧向推力；
4. 假设货物与车体相对固定，无晃动或是相对位移；
5. 非均一地面和变摩擦特性。

## A. 运动学模型

在车体坐标系 xoy 下，车辆质心的速度矢量为：

**公式 (2.1)**：
```
v⃗ = [v·cos(β), v·sin(β)]^T
```

在大地坐标系 XOY 下，位置更新方程为：

**公式 (2.2)**：
```
dX/dt = v·cos(ψ + β)
dY/dt = v·sin(ψ + β)
dψ/dt = ω
```

---

## B. 刚体动力学模型

基于牛顿－欧拉方程，考虑 3-DOF 平面运动（纵向、横向、横摆）。

### 1）纵向运动学

沿车体 x 轴的力平衡方程：

**公式 (2.3)**：
```
m_eff · a = sum(F_x) - F_drag - F_slope
```

其中：

- **总驱动力**：`sum(F_x) = F_x,lf + F_x,rr`
- **总阻力**：`F_drag = F_roll + F_aero`

```
F_roll = C_r · m · g · cos(θ_slope)
F_aero = 0.5 · ρ · C_d · A · v² · sign(v)
```

- **有效质量**：`m_eff = m + 2·(I_w + I_m·n²) / r²`

### 2）侧向动力学与侧偏角

沿车体 y 轴的力平衡：

**公式 (2.4)**：
```
sum(F_y) = m · v · (dβ/dt + ω)
```

整理得到侧偏角变化率：

**公式 (2.5)**：
```
dβ/dt = (F_y,lf + F_y,rr) / (m·v) - ω
```

> 注：公式 (2.5) 在 v → 0 时存在奇异性，物理上对应静止时侧偏角无意义。

### 3）横摆动力学

绕质心 z 轴的力矩平衡方程：

**公式 (2.6)**：
```
I_z · dω/dt = sum(M_z)
```

总力矩由两部分组成：

- **轮胎侧向力产生的力矩**：`M_z,tire = (L/2) · (F_y,lf - F_y,rr)`
  （此处假设转向角较小，侧向力主要贡献于横摆）

- **差动驱动产生的力矩**：`M_z,drive = (W/2) · (F_x,lf - F_x,rr)`

---

## C. 轮胎模型

采用线性化轮胎模型结合摩擦圆限制。

**轮胎侧偏角** - 公式 (2.7)：
```
α_lf = β + (L/2)·ω / v - δ_lf
α_rr = β - (L/2)·ω / v - δ_rr
```

**侧向力** - 公式 (2.8)：
```
F_y,lin = -C_α · α
```

考虑到附着极限 μN，实际侧向力为：

**公式 (2.9)**：
```
F_y = sign(F_y,lin) · min(|F_y,lin|, μN)
```

# II. 底层运动控制

## A. 偏航运动控制

物理模型表明差动驱动力矩 M_z,drive 取决于 F_x,lf 和 F_x,rr 的差值。模型中内置了一个 P 控制器来主动调节这个差值。

**控制目标**：使实际横摆角速度 ω 跟踪指令 ω_cmd。

**控制律**：

1. 计算误差：`e_ω = ω_cmd - ω`
2. 比例控制计算期望角加速度：`dω_des/dt = K_p · e_ω`
3. 计算所需额外力矩：`M_z,req = I_z · dω_des/dt`
4. 转换为驱动力差指令：`ΔF_x = 2·M_z,req / W`

**实现效果**：这使得 AGV 表现为一个对横摆角速度指令闭环的系统，而非开环系统。

---

## B. 稳定性增强策略

为了解决刚体模型在低速下的数值奇异性以及增强动态稳定性，模型引入了虚拟阻尼项。

### 1）虚拟侧偏阻尼

在 dβ/dt 方程中增加强阻尼项：

**公式 (2.10)**：
```
dβ/dt (model) = sum(F_y) / (m·v) - ω - 5·β
```

**物理意义**：这是一项并不存在的物理力，但在控制上等效于通过微调转向或驱动力来施加一个"使其回到运动方向"的恢复力矩。

**作用**：防止侧偏角 β 在仿真中发散，特别是在低速段，强制 β 收敛到 0，模拟了轮胎低速下的强自对准特性。

### 2）虚拟横摆阻尼

在横摆动力学中添加了与 ω 成正比的阻尼力矩：

**公式 (2.11)**：
```
M_z,total = M_z,phys - C_yaw · ω
```

**作用**：增加系统的阻尼比，减少超调和震荡，模拟车身摩擦或未建模的阻尼效应。

---

## C. 牵引力控制

模型中内嵌了基于摩擦椭圆的牵引力限制器：

**公式 (2.12)**：
```
F_x,allow = μN · sqrt(1 - F_y² / (μN)²)
```

**策略**：当检测到侧向力 F_y 较大（如急转弯）时，主动限制纵向驱动力 F_x 的上限。

**作用**：防止"顾此失彼"，确保轮胎合力不超出附着极限圆，优先保证侧向稳定性，其次提供纵向驱动力。

---

## D. 低速与静止处理

当 v < v_thresh 时：

1. **运动学**：使用显式的强阻尼 `dβ/dt = -C_low · β` 强制侧偏角归零。
2. **动力学**：禁止差动偏航控制，防止在原地产生过大的自旋力矩导致数值不稳定。
3. **阻力**：滚动阻力计算考虑了符号函数平滑，避免零速震荡。

---

## 模型总结

该 AGV 模型是一个**面向控制的增强型动力学模型**。

- **物理层**：通过牛顿－欧拉方程和非线性轮胎模型，准确描述了车辆的基本物理特性（惯性、摩擦、几何约束）。

- **控制层**：内嵌了差动偏航控制、虚拟阻尼稳定和牵引力保护，使得该模型不仅反映了"车是什么"，还反映了"底层驱动器怎么控制车"。

这种设计非常适合用于上层算法（如路径规划、轨迹跟踪 MPC）的开发与验证，因为它屏蔽了底层电机控制的复杂性，提供了一个稳定、响应可预测的被控对象接口。

# III. 控制算法

## A. 跟踪误差定义

在路径跟踪任务中，控制目标是将车辆当前的运动状态收敛至参考路径上。

![路径坐标系示意图](https://cdn.mathpix.com/cropped/551fd2e0-5d5a-4c8a-9fbe-bfa0caeb6061-5.jpg?height=568&width=812&top_left_y=495&top_left_x=463)

车辆在全局坐标系下的状态被转换为路径坐标系下的跟踪误差：

- **(X, Y)**：车辆重心在全局坐标系下的位置
- **ψ**：车辆当前的航向角
- **v**：车辆当前的纵向速度
- **ω**：车辆当前的横摆角速度
- **(X_r, Y_r)**：参考路径上正交投影点的坐标
- **ψ_r**：参考点的切向航向角
- **κ_r**：参考点的路径曲率
- **V_ref**：参考速度

---

车辆与参考点在全局坐标系下的位置偏差：

**公式 (3.1)**：
```
e_X = X - X_r
e_Y = Y - Y_r
```

为了得到相对于路径的横向误差，需要将全局位置偏差投影到参考路径的法线方向，根据旋转变换矩阵：

**公式 (3.2)**：
```
┌ e_x^path ┐   ┌  cos(ψ_r)   sin(ψ_r) ┐ ┌ e_X ┐
│          │ = │                      │ │     │
└ e_y^path ┘   └ -sin(ψ_r)  cos(ψ_r)  ┘ └ e_Y ┘  
```

其中 e_x^path 代表车辆位置相对于参考点的纵向距离。由于我们将参考点 (X_r, Y_r) 定义为车辆在路径上的正交投影点，根据正交投影的几何性质，车辆与该点的连线向量必然垂直于该点的切向向量。因此，纵向误差 e_x^path 在理论上恒为零，故而在后续控制模型中将其忽略，仅保留横向误差：

**公式 (3.3)**：
```
e_y = -sin(ψ_r)·(X - X_r) + cos(ψ_r)·(Y - Y_r)
```

航向误差定义为车辆当前航向与参考路径切向的偏差。为了消除角度周期性的影响，计算结果需归一化至主值区间：

**公式 (3.4)**：
```
e_ψ = ψ - ψ_r,  e_ψ ∈ (-π, π]
```

纵向速度跟踪误差直接定义为：

**公式 (3.5)**：
```
e_v = v - v_ref
```

为了提高曲线跟踪性能，引入基于路径曲率的前馈项。参考横摆角速度由参考速度和路径曲率由运动学关系决定：

**公式 (3.6)**：
```
ω_ref = v_ref · κ_r
```

角速度误差则定义为测量值与参考值的偏差：

**公式 (3.7)**：
```
e_ω = ω - ω_ref
```

根据上述对误差变量的分析，可以确定对 AGV 行驶的控制任务在物理上可解耦为**纵向动力**和**横向运动**两个维度的调节。具体而言，速度误差 e_v 的消除主要依赖于纵向驱动／制动力的调节，而横向位置误差 e_y 与航向误差 e_ψ 的修正则需通过精确控制车辆的转向角速度来实现。

## B. 自适应模型预测控制算法

传统线性模型预测控制（Linear MPC）在工业应用中广泛成功，其优势在于：可显式处理约束、通过滚动优化预测未来行为，且在线求解通常可转化为二次规划（QP）以获得较高计算效率。但传统线性MPC往往基于单一工作点的线性时不变（LTI）模型，性能对模型失配较为敏感。

对于自动驾驶 AGV，动力学在变曲率弯道、变坡度以及载荷/摩擦变化时呈现明显非线性与时变特性，固定 LTI 模型难以覆盖全工况。一个直接的改进途径是非线性MPC（NMPC），即在预测域内直接采用非线性模型进行滚动优化，从而在更大工况范围内保持较高模型一致性；但 NMPC 通常需要在线求解非线性规划问题，计算开销更高，且可能对初值、模型精度与实时算力更敏感。

本文采用折中且工程可实现的方案：基于线性变参数（LPV）框架构建自适应MPC。其核心思想是引入调度变量 ρ_k，在每个采样时刻根据实时运行工况对预测模型（以及权重/约束）进行在线更新，使系统在保持线性MPC求解效率与约束处理便利性的同时，获得对工况漂移的自适应能力。

为在多工况下实现一致的闭环跟踪性能，本研究构建了基于 LPV 框架的自适应模型预测控制器。其核心思想是引入**调度变量 ρ_k**，在每个采样时刻对预测模型与控制权重进行在线更新，从而以线性模型簇近似描述原系统在当前工作区间内的局部动力学。

---

### 1）LPV 预测模型与测量扰动通道

控制器的离散时间预测模型采用误差状态空间形式。定义路径坐标系下的误差状态向量为：

**公式 (3.8)**：
```
x_k = [e_y, e_ψ, e_v, e_ω]^T
```

测量扰动为坡度角的在线估计值：

**公式 (3.9)**：
```
d_k = θ_k
```

在采样时刻 k，预测模型写为：

**公式 (3.10)**：
```
x_{k+1} = A(ρ_k)·x_k + B(ρ_k)·u_k + E(ρ_k)·d_k
y_k = C(ρ_k)·x_k
```

其中 y_k 为 MPC 的测量输出。

调度变量选取为 **ρ_k = [v_k, ω_k, θ_k]^T**，其中：
- v_k 与 ω_k 来自车辆状态／参考量
- θ_k 可由上层 GRU 模块提供在线估计 θ̂_k

为抑制调度跳变带来的数值不稳定性，本文对 ρ_k 采用**一阶低通滤波**得到 ρ_k^f，并以 ρ_k^f 作为模型更新的实际调度输入。

具体实现中，矩阵 A(ρ), B(ρ), E(ρ) 来自三维网格线性化数据库，并通过**三线性插值**计算。其等价形式可表示为 8 个网格顶点的凸组合：

**公式 (3.11)**：
```
A(ρ) = Σ_{j=1}^{8} w_j(ρ)·A_j
B(ρ) = Σ_{j=1}^{8} w_j(ρ)·B_j
E(ρ) = Σ_{j=1}^{8} w_j(ρ)·E_j
```

其中 $w_j(\rho) \geq 0$ 且 $\sum_j w_j(\rho) = 1$。

#### 系统矩阵结构

对于路径坐标系误差模型，离散状态空间矩阵具有如下结构。

状态矩阵 $\mathbf{A}(\rho) \in \mathbb{R}^{4 \times 4}$ 定义为：

$$
\mathbf{A}(\rho) = \begin{bmatrix} A_{11} & A_{12} & 0 & 0 \\ 0 & A_{22} & 0 & A_{24} \\ 0 & 0 & A_{33} & 0 \\ 0 & 0 & 0 & A_{44} \end{bmatrix} \tag{3.12}
$$

各元素由下式给出：

$$
A_{11} = 1
$$

$$
A_{12} = v \cdot T_s
$$

$$
A_{22} = 1
$$

$$
A_{24} = T_s
$$

$$
A_{33} = 1 - \frac{C_r g + \rho C_d A v}{m_{eff}} T_s
$$

$$
A_{44} = 1 - \frac{C_{yaw}}{I_z} T_s
$$

其中 $C_{yaw}$ 为横摆阻尼系数。

输入矩阵 $\mathbf{B}(\rho) \in \mathbb{R}^{4 \times 2}$ 定义为：

$$
\mathbf{B}(\rho) = \begin{bmatrix} 0 & 0 \\ 0 & 0 \\ B_{31} & 0 \\ 0 & B_{42} \end{bmatrix} \tag{3.13}
$$

各元素由下式给出：

$$
B_{31} = \frac{T_s}{m_{eff}}
$$

$$
B_{42} = \frac{K_\omega T_s}{I_z}
$$

其中 $K_\omega$ 为底层角速度控制器的比例增益。

输出矩阵 $\mathbf{C} \in \mathbb{R}^{4 \times 4}$ 采用单位阵形式：

$$
\mathbf{C} = \mathbf{I}_4 \tag{3.14}
$$

扰动矩阵 $\mathbf{E}(\rho) \in \mathbb{R}^{4 \times 1}$ 定义为：

$$
\mathbf{E}(\rho) = \begin{bmatrix} 0 \\ 0 \\ E_3 \\ 0 \end{bmatrix} \tag{3.15}
$$

其中：

$$
E_3 = -\frac{g \cdot T_s}{m_{eff}}
$$

该项反映坡度角 $\theta$ 对纵向速度误差的直接影响。

> **注**：上述矩阵元素均为调度变量 $\rho = [v, \omega, \theta]^\top$ 的函数。实际实现中，通过在三维网格各节点处对非线性模型进行数值线性化，获得离散矩阵族 $\{A_j, B_j, E_j\}$，并在运行时通过三线性插值计算当前工作点处的矩阵值。

### 2）滚动时域优化问题

在每个采样时刻 k，控制器求解有限预测域内的滚动优化问题。设预测步长为 N_p、控制步长为 N_c，以控制增量序列 ΔU_k = {Δu_{k|k}, Δu_{k+1|k}, ..., Δu_{k+N_c-1|k}} 为决策变量，最小化如下二次型代价：

**公式 (3.12)**：
```
\[\begin{align}
  & {\min_{\Delta U_k}}\, J_k = \sum\limits_{i=1}^{N_p} \left( y_{k+i|k}-r_{k+i|k} \right)^{T} Q\left(\rho_k^{f}\right) \left( y_{k+i|k}-r_{k+i|k} \right) \\
  & + \sum\limits_{i=0}^{N_c-1} u_{k+i|k}^{T} R\left(\rho_k^{f}\right) u_{k+i|k}
    + \sum\limits_{i=0}^{N_c-1} \left( \Delta u_{k+i|k} \right)^{T} dR\left(\rho_k^{f}\right) \left( \Delta u_{k+i|k} \right) \\
  & + \sum\limits_{i=1}^{N_p} e_{slack\,,k+i|k}^{T} W_{slack} e_{slack\,,k+i|k}.
\end{align}\]
```

其中 $Q(\rho)$、$R(\rho)$、$dR(\rho)$ 分别对应输出误差、操纵变量及操纵变量变化率的权重。本研究在误差坐标系下采用 $r_{k+i|k}=0$ 作为参考，从而直接驱动误差收敛。为提高可行性，本文对关键输出误差（如 $e_y$、$e_\psi$）采用软约束，并通过松弛变量 $e_{slack}$ 引入额外惩罚项；其权重 $W_{slack}$ 对应实现中的软约束惩罚系数（MinECR/MaxECR）。

约束条件包括操纵变量幅值与变化率约束，以及输出误差约束：

**公式 (3.13)**：
```
x_{k+i+1|k} = A(ρ_k^f)·x_{k+i|k} + B(ρ_k^f)·u_{k+i|k} + E(ρ_k^f)·d_{k+i|k}
u_min(ρ_k^f) ≤ u_{k+i|k} ≤ u_max(ρ_k^f)
Δu_min(ρ_k^f) ≤ Δu_{k+i|k} ≤ Δu_max(ρ_k^f)
```

并对关键输出误差设置**软约束**以提高可行性。扰动项 d_{k+i|k} 在实现中使用当前测量扰动输入并在预测域内采用冻结近似。

---

### 3）在线自适应更新机制

本文的自适应机制在每个采样时刻执行如下步骤：

1. 由车辆状态／参考量与环境估计构造并滤波调度变量 ρ_k^f = [v_k, ω_k, θ_k]^T；
2. 由 LPV 数据库对 A, B, C, D, E 做三线性插值，得到当前时刻预测模型；
3. 根据 ρ_k^f 对权重 Q, R, dR 以及输入约束上下界进行在线调度；
4. 将上述更新后的模型与参数输入 MPC 求解器，计算当前最优控制量 u_k 并施加至被控对象。

通过以上机制，控制器能够在**模型层**与**优化层**同时反映当前工况变化，从而提升系统在转弯与坡度扰动等复杂工况下的跟踪性能与鲁棒性。

## C. GRU算法

在 AGV 的坡道与转弯工况下，车辆纵向动力学会受到坡度角 $\theta$ 的显著影响，使得误差模型中的扰动项 $d_k=\theta_k$ 以及调度变量 $\rho_k=[v_k,\omega_k,\theta_k]^T$ 随工况快速变化。如果仅依赖固定阈值或简单几何关系从 IMU 直接反推出坡度，往往会受到非重力加速度、轮胎振动以及噪声漂移的干扰，导致 $\theta$ 估计抖动并进一步引起 LPV 插值与权重/约束调度不稳定。为此，本文在控制器上层引入 GRU（Gated Recurrent Unit）作为序列特征提取器，利用车辆可测输出构造时间序列输入 $x_{t}$，在线输出坡度角估计 $\hat{\theta}$ 并注入 MPC 的测量扰动通道（MD），从而实现更平滑且抗噪的坡度感知。项目中 $x_t$ 由 Simulink 输出向量 $y_{raw}(31\times 1)$ 的若干通道及其派生量构成（本版本实际使用 19 维特征），主要包含 IMU 加速度/角速度、轮速、舵角、电机电流与速度差分等信息；坡度真值仅作为监督标签，不作为网络输入。

选择 GRU 的原因在于其在保持时序建模能力的同时具备更低的参数规模与推理开销，更适合实时控制回路集成。相比基于 IMU 的传统坡度估计（如加速度重力分解、积分或互补滤波），GRU 能在存在机体纵向加减速、路面扰动和传感器偏置时，利用多源量测的时序一致性学习“坡度—动力响应—量测特征”的隐式映射，从而降低由非重力加速度引起的系统性误差。相比 LSTM，GRU 采用更简化的门控结构，在相近精度下通常具有更快的收敛速度与更低的计算/存储占用。其核心状态更新可写为：

**公式 (3.14)**：
```
\[\begin{align}
  r_t &= \sigma\left(W_r x_t + U_r h_{t-1} + b_r\right), \\
  z_t &= \sigma\left(W_z x_t + U_z h_{t-1} + b_z\right), \\

  \widetilde{h}_t &= \mathrm{tanh}\left(W_h x_t + U_h\left(r_t\odot h_{t-1}\right) + b_h\right), \\
  h_t &= \left(1-z_t\right)\odot h_{t-1} + z_t\odot \widetilde{h}_t.
\end{align}\]
```

其中 $r_t$ 为重置门（Reset Gate），$z_t$ 为更新门（Update Gate），$\widetilde{h}_t$ 为候选隐状态（Candidate Hidden State），$h_t$ 为时刻 $t$ 的隐状态（Hidden State）。$x_t$ 表示输入特征向量，$W_r,W_z,W_h$ 与 $U_r,U_z,U_h$ 为可训练权重矩阵，$b_r,b_z,b_h$ 为偏置项；$\sigma(\cdot)$ 与 $\tanh(\cdot)$ 分别为 Sigmoid 与双曲正切非线性函数，$\odot$ 表示按元素乘（Hadamard 乘积）。

**公式 (3.15)**：
```
\[\begin{align}
  p^{main}_t &= \mathrm{softmax}\left(W_{main} h_t + b_{main}\right), \\
  p^{turn}_t &= \mathrm{softmax}\left(W_{turn} h_t + b_{turn}\right), \\
  \hat{\theta}_t &= W_{\theta} h_t + b_{\theta}.
\end{align}\]
```

公式 (3.15) 给出了 GRU 的三路输出形式：$p_t^{main}$ 与 $p_t^{turn}$ 分别为主工况与转弯状态的分类概率向量（Softmax 输出，预测类别可取 $\arg\max p_t$，其最大分量可作为置信度指标）；$\hat{\theta}_t$ 为坡度角的连续值回归输出，用于向 MPC 提供在线扰动/调度量。若 $h_t\in\mathbb{R}^{H}$，主工况类别数为 $C_{main}$、转弯类别数为 $C_{turn}$，则 $W_{main}\in\mathbb{R}^{C_{main}\times H}$、$W_{turn}\in\mathbb{R}^{C_{turn}\times H}$，而 $W_{\theta}\in\mathbb{R}^{1\times H}$（对应标量坡度回归）。

结合项目实际训练与评估结果：当前版本训练采用序列长度 $seq\_len=48$、步幅 $stride=12$，GRU 隐层规模 96、两层结构与 $dropout=0.2$，最大训练轮次 200，并在第 134 个 epoch 早停收敛。离线数据集划分为 Train/Val/Test（序列样本数分别为 72615/15031/15054），在 Test 集上主工况分类准确率达到 0.9903、转弯分类准确率 0.9926，坡度角回归的 MAE 为约 $1.08^\circ$。在线评估共 60 段场景（每段 20 s），主工况识别平均准确率为 0.9677；在包含坡度变化的 20 段场景中，坡度估计 MAE 平均约 $0.86^\circ$，坡度进入/切换的平均识别延迟约 1.335 s。进一步在闭环仿真评估（6 类场景，每类 10 次，共 60 次）中，速度跟踪误差的均方根（RMS）均值为 0.0303，坡度估计 MAE 均值约 $0.31^\circ$，并在长上坡等典型坡度场景中保持稳定的坡度识别与控制调度效果。

## D. 使用贝叶斯优化进行控制器调参

LPV-MPC 控制器涉及权重矩阵、约束边界、调度滤波以及工况自适应阈值等多类超参数，且参数之间存在显著耦合。若采用手动调参，不仅需要大量闭环仿真迭代、时间成本高，而且高度依赖控制工程经验；在多场景（转弯、颠簸、坡道、直线等）共同约束下，手动方法往往只能获得次优折中解。为此，本文采用贝叶斯优化（Bayesian Optimization, BO）将闭环评估过程视为黑盒目标函数，通过最小化综合代价 $J(\phi)$ 自动搜索控制器参数向量 $\phi$，实现可复现、可扩展的自动化调参。

贝叶斯优化适用于“单次评估代价高、无解析梯度且目标可能非凸”的超参数搜索问题。本文将调参建模为两层优化结构：内层在给定 $\phi$ 时执行闭环仿真并计算代价 $J(\phi)$；外层通过代理模型与采集函数选择下一组参数进行评估。其形式可写为：

**公式 (3.16)**：
```
\[\begin{align}
  \phi^{\star} &= \arg\min_{\phi\in\Phi} f(\phi), \\
  f(\phi) &= J(\phi) = \sum_{s\in\mathcal{S}} w_s\, J_s(\phi).
\end{align}\]
```

其中 $\phi^{\star}$ 表示使目标函数达到最小的最优参数向量（即 BO 最终返回的“最优调参结果”）。$\mathcal{S}$ 为场景集合，$w_s$ 为场景权重。为体现 $J_s(\phi)$ 与闭环性能指标的“具体关系”，本文在实现中将 $J_s(\phi)$ 构造为若干可计算指标的加权组合，例如：

```
\[\begin{align}
  J_s(\phi) &= \lambda_y\,\mathrm{RMSE}(e_y) + \lambda_{\psi}\,\mathrm{RMSE}(e_{\psi}) + \lambda_v\,\mathrm{RMSE}(e_v) + \lambda_{\omega}\,\mathrm{RMSE}(e_{\omega}) \\
  &\quad + \lambda_{\Delta u}\,\mathrm{RMSE}(\Delta u) + \lambda_{\mathrm{vio}}\,\Pi_{\mathrm{vio}} + \lambda_T\,\overline{T}_{\mathrm{solve}}.
\end{align}\]
```

其中 $\Pi_{\mathrm{vio}}$ 表示约束违反惩罚项（可由违反幅度/持续时间等统计得到），$\overline{T}_{\mathrm{solve}}$ 表示平均求解耗时。外层 BO 在第 $n$ 次迭代基于历史观测数据 $\mathcal{D}_n=\{(\phi_i,f(\phi_i))\}_{i=1}^{n}$ 拟合代理模型（通常为高斯过程）并通过采集函数 $a(\phi;\mathcal{D}_n)$ 权衡探索与利用，选择下一次评估点：

**公式 (3.17)**：
```
\[\begin{align}
  \phi_{n+1} = \arg\max_{\phi\in\Phi} a\left(\phi;\mathcal{D}_n\right).
\end{align}\]
```

具体实现中，本文采用两阶段流程：第一阶段进行全局搜索获得较优解；第二阶段以第一阶段最优点为中心收缩边界进行局部精细搜索，以提升收敛质量并降低遗漏较优解的风险。

贝叶斯优化适用于本文调参任务的原因在于：其样本效率高（适合昂贵仿真）、不依赖梯度（适合含饱和/切换/软约束的非光滑闭环评估），且目标构造灵活（只需修改 $J(\phi)$ 的组合方式即可重新权衡精度、平滑性、约束与实时性）。结合项目实现，BO 实际调优的参数向量 $\phi$ 主要包括：输出误差权重基值 $q_y,q_{\psi},q_v,q_{\omega}$；操纵量权重 $r_F,r_{\omega}$ 与增量权重 $d r_F, d r_{\omega}$（实现中采用 $\log_{10}$ 形式优化以覆盖数量级变化）；权重随工况变化的形状参数 $\alpha_Q,\beta_Q,\alpha_R,\beta_R,\alpha_{dR},\beta_{dR}$；输入/增量约束的缩放系数 $scale\_{umin},scale\_{umax}$（含 lo/hi 区间缩放）；调度量滤波时间常数 $\tau$；以及场景自适应参数（如转弯触发阈值与过渡宽度、横向权重增益上限；坡度触发阈值与过渡宽度、纵向权重增益上限，以及上坡/下坡时 $R_F$ 与 $dR_F$ 的增益上限等）。优化完成后生成最优参数映射并用于后续闭环验证。

## A. GRU 学习和优化结果

本节首先给出 GRU 的学习结果与泛化表现，并进一步结合在线推理环节中的“延迟敏感参数”说明其工程化优化要点。GRU 采用“主工况分类 + 转弯状态分类 + 坡度角回归”的多任务输出结构，以序列输入刻画车辆在坡道/转弯工况下的动力学特征。网络输入由闭环平台可测的 31 维原始观测通道及其派生量构成（最终选取 19 维特征）；训练采用序列长度 48（约 2.4 s）与步幅 12（约 0.6 s），网络为两层 GRU、隐层规模 96，并设置 $dropout=0.2$。训练最大轮次 200，并在第 134 个 epoch 触发早停。离线数据集划分为 Train/Val/Test（序列样本数分别为 72615/15031/15054），用于评估模型的收敛与泛化能力。

为保证训练数据的可复现性与可追溯性，本文所用 GRU 数据集由“仿真生成—预处理切片”两阶段流程得到。第一阶段在 Simulink 闭环仿真环境下批量生成多场景时序数据：通过对参考路径与初始条件进行随机化，并在其中 70% 的回合引入传感器噪声注入，同时可施加牵引效率退化、外载荷突变等外部扰动以提升分布覆盖，从而获得包含不同工况切换与扰动强度的样本。每个回合记录时间序列、控制输入 $u=[F_{cmd},\,\omega_{cmd}]$、31 维原始观测以及自动标注标签，其中主工况分类采用三分类设置（平地、堵转、坡度），转弯状态分类采用三分类设置（左转、直行、右转），并同时提供坡度角真值 $\theta$ 作为回归监督信号。第二阶段对原始观测提取“仅由可测原始量与简单派生量构成”的 19 维特征，并采用滑窗形成序列样本：在每个回合内部按固定步幅递推截取长度为 48 的子序列（步幅 12，因此相邻样本存在重叠），且仅保留完整窗口而不进行零填充；监督信号采用“窗口末端对齐”的定义，即主工况/转弯标签与坡度真值均取窗口最后一个采样时刻的标注值，从而使每个样本可解释为“利用过去约 2.4 s 的量测预测当前时刻的工况与坡度”。数据划分按回合分组以降低跨回合泄漏风险，并按 0.70/0.15/0.15 的比例划分训练集/验证集/测试集；归一化采用 z-score 且仅使用训练集统计量。需要强调的是，坡度真值 $\theta$ 仅作为监督标签参与训练与离线评估，不作为网络输入特征。

在训练设置方面，本文采用基于自定义训练循环的 Adam 优化器，初始学习率设为 $1\times 10^{-3}$，并使用余弦退火进行学习率衰减：
$$
\eta(e)=\eta_0\cdot\frac{1}{2}\left(1+\cos\left(\pi\frac{e}{E_{\max}}\right)\right),
$$
其中 $e$ 为当前 epoch，$E_{\max}=200$ 为最大训练轮次，批量大小为 64。训练过程中采用全局范数梯度裁剪（阈值 5.0）以抑制梯度爆炸，并基于验证集总损失实施早停：当验证损失在连续 20 个 epoch 内未能超过 $10^{-4}$ 的改善幅度时终止训练，并回退到验证损失最优时刻的模型参数。多任务损失由主工况加权交叉熵、转弯交叉熵以及坡度回归损失组合构成，其中坡度回归误差仅在“坡度工况”样本上计入，以避免平地样本对回归项产生不必要的偏置。

在评估口径方面，离线评估在 Train/Val/Test 三个切分上分别统计主工况与转弯状态的分类准确率（逐样本平均），并计算坡度回归的平均绝对误差（MAE）；其中 MAE 仅在真值属于坡度工况的样本集合上统计。在线评估采用逐采样点的序列推理方式，为避免序列缓冲未填满导致的冷启动效应，评估从序列长度之后的时刻开始统计分类准确率与坡度 MAE。坡度识别延迟定义为真值首次进入坡度工况与预测首次进入坡度工况之间的时间差，用于刻画工况切换时的响应滞后。

离线测试集上，主工况分类准确率达到 0.9903、转弯分类准确率 0.9926；坡度角回归在测试集上的 MAE 约为 $1.08^\circ$。上述结果表明网络能够在多源量测噪声与非线性扰动存在的条件下学习到与坡度/工况相关的时序特征，从而在“分类（离散工况）”与“回归（连续坡度）”两个维度同时获得较高精度。

为验证在线可用性，本文在 60 段在线场景中进行推理评估（每段 20 s）。在线主工况识别平均准确率为 0.9677；在包含坡度变化的 20 段场景中，坡度估计 MAE 平均约 $0.86^\circ$，坡度进入/切换的平均识别延迟约 1.335 s。进一步在闭环仿真评估（6 类场景，每类 10 次，共 60 次）中，速度跟踪误差的均方根（RMS）均值为 0.0303，坡度估计 MAE 均值约 $0.31^\circ$，说明在闭环条件下（控制输入抑制了部分高频扰动并使工况更可预测）坡度估计可获得更稳定的统计表现。

在工程部署层面，在线推理的可用性不仅取决于网络本体，还受到“序列缓冲/驻留时间/低通滤波”等后处理环节影响。本文在在线推理端对主工况与转弯状态引入最小驻留时间（分别为 0.20 s 与 0.40 s）以抑制标签抖动，并对坡度估计 $\hat{\theta}$ 采用一阶低通滤波（时间常数 $\tau_{\theta}=0.15\ \mathrm{s}$）以降低噪声引起的调度跳变。此外，为进一步抑制平地工况的微小估计偏差，输出端引入坡度死区（约 $1.3^\circ$ 的硬死区与 $1.8^\circ$ 的软死区）。上述参数通过延迟敏感性评估（控制变量扫描 + 组合验证）确定，目标是在保持识别准确率的前提下降低识别延迟与状态抖动，从而使 GRU 输出更适合注入 MPC 的扰动/调度通道。
